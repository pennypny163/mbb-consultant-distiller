## 一、市场规模预测框架

### 1.1 渗透率分解法（Top-Down）

```
市场规模 = 目标客群 × 渗透率 × 客单价 × 频次
```

**示例**：中国健身房市场
```
市场规模 = 城镇人口(8.5亿) × 健身渗透率(3%) × 年均消费(3000元)
         ≈ 765亿元
```

### 1.2 品类分解法

按产品/服务类型拆分：
```
市场规模 = 品类A规模 + 品类B规模 + 品类C规模
```

### 1.3 预测方法

| 方法 | 适用场景 | 示例 |
|------|---------|------|
| 回归分析 | 有强相关变量 | GDP增速 → 行业增速 |
| 对标法 | 成熟市场参照 | 美国经验 → 中国预测 |
| 专家访谈 | 缺乏数据时 | 行业协会访谈 |

## 二、竞争格局分析框架

### 2.1 波特五力

1. **现有竞争者** - 数量、集中度、差异化程度
2. **潜在进入者** - 壁垒高低、资本要求
3. **替代品威胁** - 功能替代程度
4. **供应商议价能力** - 集中度、替代成本
5. **买方议价能力** - 数量、集中度、信息透明度

### 2.2 竞争者分析矩阵

| 维度 | 竞争者A | 竞争者B | 竞争者C |
|------|--------|--------|--------|
| 市场份额 | 30% | 25% | 15% |
| 增速 | 15% | 20% | 8% |
| 定价 | 高 | 中 | 低 |
| 渠道覆盖 | 全国 | 区域 | 线上 |
| 核心优势 | 品牌 | 价格 | 技术 |

### 2.3 关键成功要素 (KSF)

```
KSF = 行业成功必须的3-5项核心能力

示例 - 健身房行业KSF：
1. 选址（位置/人流）
2. 定价策略
3. 会员服务体验
4. 教练团队
5. 设备设施
```

## 三、市场进入判断框架

### 3.1 是否进入 (Market Entry)

```
┌─────────────────────────────────────────────────────────┐
│                    外部分析 (External)                   │
├─────────────────────────────────────────────────────────┤
│  市场规模 vs 现有业务  │  利润水平对比                  │
│  行业增速 vs 现有业务  │  监管政策分析                  │
│  竞争格局        │  政策限制/牌照                     │
├─────────────────────────────────────────────────────────┤
│                    内部分析 (Internal)                   │
├─────────────────────────────────────────────────────────┤
│  管理团队意愿    │  协同效应分析                      │
│  核心能力匹配    │  资源投入需求                       │
└─────────────────────────────────────────────────────────┘
                          ↓
                    进入方式选择：
              自建 / 收购 / 合资 / 战略合作
```

### 3.2 进入方式比较

| 维度 | 自建 | 收购 | 合资 | 战略合作 |
|------|------|------|------|----------|
| 话语权 | 高 | 高 | 中 | 低 |
| 速度 | 慢 | 快 | 中 | 快 |
| 资源投入 | 高 | 高 | 中 | 低 |
| 风险 | 高 | 中 | 中 | 低 |

## 四、转型战略框架 (Turn Around)

### 4.1 分析层次

```
1. 宏观判断：市场对目标公司是否还有吸引力？
2. 细分市场：哪些细分赛道值得投入？
3. 客户分析：目标客群是谁？核心诉求？
4. 竞争分析：谁做得好？为什么？
5. 能力建设：需要建立什么核心能力？
```

### 4.2 细分市场评估

| 维度 | 权重 | 细分市场A | 细分市场B |
|------|------|----------|----------|
| 市场规模 | 25% | ★★★★☆ | ★★★☆☆ |
| 增速 | 25% | ★★★★★ | ★★★★☆ |
| 利润率 | 20% | ★★★☆☆ | ★★★★★ |
| 竞争激烈度 | 15% | ★★★☆☆ | ★★☆☆☆ |
| 政策支持 | 15% | ★★★★☆ | ★★☆☆☆ |
| **加权得分** | 100% | **3.9** | **3.7** |

## 五、TMT/新技术分析框架

### 5.1 分析路径

```
┌─────────────────────────────────────────────────────────┐
│  Step 1: 应用场景识别                                     │
│  → 这个技术可以用在哪些场景？                              │
├─────────────────────────────────────────────────────────┤
│  Step 2: 市场规模测算                                     │
│  → 每个场景的市场规模？成长拐点？                          │
├─────────────────────────────────────────────────────────┤
│  Step 3: 产业链分析                                       │
│  → 上中下游有哪些环节？核心壁垒在哪？                       │
├─────────────────────────────────────────────────────────┤
│  Step 4: 商业模式比较                                     │
│  → 软硬一体 vs 分工模式？各阶段哪种更优？                   │
├─────────────────────────────────────────────────────────┤
│  Step 5: 竞争格局推演                                     │
│  → 科技公司/主机厂/平台商，谁主导？                        │
└─────────────────────────────────────────────────────────┘
```

### 5.2 产业格局分析

三类玩家竞争主导权：
1. **科技公司** - 算法/系统能力
2. **传统企业** - 制造/渠道能力
3. **平台公司** - 用户/运营能力

## 七、金字塔原理应用

### 7.1 单页PPT结构

```
┌─────────────────────────────────────────────────────────┐
│  Title: [核心观点/结论]                                    │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  Key Message: [一句话说明本页要传达的核心信息]              │
│                                                          │
├─────────────────────────────────────────────────────────┤
│  支撑论据A ─┬─ 事实/数据1                                 │
│             ├─ 事实/数据2                                 │
│             └─ 事实/数据3                                 │
│                                                          │
│  支撑论据B ─┬─ 事实/数据1                                 │
│             ├─ 事实/数据2                                 │
│                                                          │
│  支撑论据C ─┬─ 事实/数据1                                 │
│             └─ 事实/数据2                                 │
└─────────────────────────────────────────────────────────┘
```

### 7.2 逻辑检查

| 检查项 | 说明 |
|--------|------|
| 结论先行 | 标题/首句是否为结论？ |
| MECE | 各部分是否相互独立、无遗漏？ |
| 数据支撑 | 是否有具体数字/事实？ |
| 逻辑链完整 | 结论→论据→事实是否连贯？ |
